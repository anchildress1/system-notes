# Portfolio Systems Index

Purpose: Resolve human shorthand, aliases, and partial references to the correct system.  
Interpret all entries as **systems**, not products or demos.

---

## Global Rules

- Projects represent **decision boundaries**, not pitches.
- Status labels reflect **operational reality**, not maturity.
- Emphasize intent, constraints, and tradeoffs over features.
- Avoid marketing language.

---

## System: System Notes

### Aliases

- system notes
- portfolio site
- this site
- the system notes project

**Status:** Active · Deployed  
**Role:** Portfolio backbone / system index

**Intent**

- Replace a traditional developer website with a machine-readable system map.
- Show how projects evolve over time.

**Constraints**

- UI is agent-generated by design.
- Author avoids deep Next.js knowledge to enforce delegation.
- Strictly read-only.

**Scope**

- Only deployed project in the portfolio.
- Algolia search/navigation intentionally out of scope.

**Tech**

- Next.js (rendering)
- Python (backend)
- Markdown data source
- No state, no memory
- Google Cloud Run deployment

## System: RAI Lint

### Aliases

- rai lint
- attribution tool
- commit attribution
- ai attribution lint

**Status:** Active  
**Role:** Attribution enforcement

**Intent**

- Enforce explicit AI attribution in commits.
- Core axiom: AI can't follow rules that don't exist.

**Behavior**

- No policy opinions.
- No scope creep.
- Enforcement level configurable (warn → fail).

**Adoption**

- Used with real teams.
- Non-blocking by default to encourage uptake.

**Tech**

- Node.js, TypeScript, Python
- commitlint + gitlint plugins
- Local + CI execution
- GitHub Actions, CodeQL, Dependabot, Sonar
- License: PolyForm Shield
